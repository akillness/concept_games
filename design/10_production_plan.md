# 10. Production Plan

## 목표

`Project Moss Harbor`를 솔로 또는 2~3인 소규모 팀이 구현 가능한 범위로 유지한다.

## 개발 단계

### Phase 1 - Prototype

- 이동
- 정화 타일
- 오염 노드
- 기본 자원 드랍
- 허브 복원 1개

완료 기준:
- 10분짜리 원정 1개가 작동
- 귀환 후 허브 변화가 보임

### Phase 2 - Vertical Slice

- 1지구 완성
- HUD, Results, Quest Board
- 기본 업그레이드 6개
- 주민 2명

완료 기준:
- 60분 플레이 가능한 작은 완성본

### Phase 3 - Content Production

- 나머지 5지구 제작
- 허브 전 구역 구현
- 장식, 주민, 엔딩 추가

### Phase 4 - Polish

- VFX
- SFX
- 접근성 옵션
- 난이도 재튜닝
- 패드 대응

## 인력별 분담

### 기획

- 목표/규칙 고정
- 지구별 체크리스트
- 경제 수치표

### 프로그래밍

- 코어 서비스
- 원정 런타임
- 허브 복원 시스템
- 저장 시스템

### 아트

- 팔레트 정의
- 허브/지구 소품 통일
- 오염 전후 머티리얼 세트

## 리스크 관리

### 리스크 1: 콘텐츠 부족

대응:
- 지구당 일반 섹터 3 + 결전 1 구조 고정
- 자유 복원 모드 후속 확보

### 리스크 2: 시스템 과확장

대응:
- 농사, 자동화, 복잡 상점 시뮬레이션 금지
- 장식은 슬롯 기반으로 유지

### 리스크 3: 에셋 불일치

대응:
- 초기에 컬러 토큰과 재질 가이드 고정
- Built-in 기준 검증 후 채택

### 리스크 4: 후반 반복감

대응:
- 지구별 목표 타입 차별화
- 모듈 조합과 허브 기능 연동

## 출시 가능 범위

### Must Have

- 튜토리얼 + 6지구 + 엔딩
- 허브 복원 5구역
- 장비 4종
- 주민 5명

### Nice to Have

- 자유 복원 모드 확장 목표
- 사진 모드
- 계절 장식 DLC 훅

### Cut First

- 음성
- 고급 NPC 스케줄링
- 다층 서브스토리
- 복잡한 도감 UI
