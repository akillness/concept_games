# 11. Task Estimation

## 목적

현재 구현 상태를 기준으로 `Project Moss Harbor`를 실제 플레이 가능한 vertical slice에서 8시간 분량의 프리미엄 완결형 빌드까지 확장하기 위한 개발 추정 문서다.

이 문서는 다음 세 가지를 동시에 만족시키는 것을 목표로 한다.

- 현재 코드베이스에 이미 있는 것과 없는 것을 분리한다.
- 남은 작업을 `13+` 포인트 에픽이 아니라 실행 가능한 스토리로 분해한다.
- 솔로 또는 2~3인 소규모 팀 기준으로 우선순위와 일정 범위를 제시한다.

## 추정 기준

### Story Point 기준

- `1`: 1~2시간, 단순 상수/문구/연결 수정
- `2`: 반나절 이하, 단일 책임의 작은 기능 추가
- `3`: 0.5~1일, 명확한 로직 추가 + 가벼운 테스트
- `5`: 1~2일, 시스템 1개 또는 UI/데이터/로직 결합 작업
- `8`: 2~3일, 복수 시스템 연동 또는 높은 검증 비용
- `13+`: 분리 필요

### 추정 가정

- 현재 구현 완료 범위:
  - `Boot -> Hub -> Expedition_Runtime -> Results -> Hub` 루프 존재
  - 6지구/6퀘스트/3허브 업그레이드 데이터 존재
  - 저장/로드, 결과 요약, 기본 HUD 존재
  - Unity MCP 기준 에디터 연결 복구 완료
  - EditMode 테스트 `45/45` 통과
  - 현재 운영 게이트 통과 (Hub/Expedition 재생, 콘솔 error `0`, UV guardrail `critical=0`)
- 현재 미구현 또는 부족 범위:
  - 실제 플레이 목표 다양화
  - 허브 복원/건설 콘텐츠
  - ScriptableObject 기반 콘텐츠 양 확장
  - VFX/SFX/튜토리얼/엔딩/접근성

## 현재 상태 요약

### 이미 구현됨

- 코어 서비스 골격
- 6지구 카탈로그 및 해금 흐름
- 자원 획득 및 결과 저장
- 허브 업그레이드 3종
- 원정 런타임 생성형 픽업
- 기본 UI/HUD

### 아직 필요한 것

- 허브 복원 구역의 실제 시각/상태 변화
- 지구별 목표 타입 차별화
- 튜토리얼/첫 30분 온보딩
- 엔딩 및 자유 복원 모드
- 콘텐츠 수치 테이블 정교화

## 기준 스토리

### 기준 스토리 A

- 스토리: `ContentPaths` 확장 + 지구/퀘스트 자산 1세트 추가
- 기준점: `3점`

### 기준 스토리 B

- 스토리: 허브 업그레이드 1종 추가 + UI 반영 + 저장 연동
- 기준점: `5점`

### 기준 스토리 C

- 스토리: 원정 결과를 허브 상태 변화로 반영하는 신규 시스템
- 기준점: `8점`

  - [ ] 신규 저장에서 튜토리얼 단계가 자동 시작된다
  - [ ] 플레이어는 첫 원정 진입 전 필요한 버튼과 목표를 이해할 수 있다
  - [ ] 튜토리얼 완료 후 일반 루프로 전환된다

## 리스크 버퍼

### 중간 리스크

- Unity 씬 상태와 런타임 생성형 UI가 충돌할 수 있음
- 허브 복원 시각 표현이 단순 텍스트에 머물 위험
- 퀘스트/해금/별점 곡선이 후반 반복감을 키울 수 있음

### 버퍼 적용

- 시스템성 태스크 `5점`은 실제 일정에서 `+20~30%` 버퍼 권장
- 새로운 씬/메뉴/엔딩 태스크는 `+1일` QA 버퍼 권장
