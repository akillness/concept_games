[Lighthouse Crown] <──(5★)── [Sunken Arcade] <──(4★)── [Glass Narrows]

**픽업 배치 좌표** (total=4, radius=7m)

| 인덱스 | 타입 | X | Z | 각도 |
|--------|------|-----|-----|------|
| 0 | B1 | +7.0 | 0.0 | 0° |
| 1 | B2 | 0.0 | +7.0 | 90° |
| 2 | B3 | -7.0 | 0.0 | 180° |
| 3 | S1 | 0.0 | -7.0 | 270° |

**배치도 (Top-down)**

```
              N (+Z)
              |
         B2 (0,+7)
        /    |
       /     |
  B3(-7,0)──[ORIGIN]──B1(+7,0)  → E (+X)
       \     |
        \    |
         S1(0,-7)
              |
         [PLAYER SPAWN] (0, 0, 0)

  BEACON: (0, 0.75, 10) — 북쪽 10m
```

**기믹 동선**
- 최적 루트 (시계방향): B1 → B2 → B3 → S1 → Beacon
- 총 이동 거리 예상: ~45m
- 스폰에서 비콘까지 직선 거리: 10m
- 환경 위험: 느린 오염 슬라임, 미끄러운 물웅덩이 (design/05_content_plan.md)
- 비콘 도달 전 3개 픽업 수집 권장 (목표: CollectPickups 3개)

**난이도별 변화**

| 모드 | 타이머 기준 | 2성 조건 (픽업 비율) | 3성 조건 (남은 시간 비율) |
|------|------------|---------------------|--------------------------|
| twoStarPickupRatio | — | 0.8 (≥80% 수집) | — |
| threeStarTimeRatio | — | — | 0.65 (65% 이상 남기기) |
| 210초 기준 3성 컷 | ~137초 내 완료 | — | — |

- 환경 위험: 긴 이동 동선, 자원 회수 우선순위, 반사 락

**기믹 동선**
- 방사형 복합 구조: 8개 픽업이 45° 간격으로 원형 배치 → 어느 방향으로도 이동 가능
- HoldOut 90초: 전체 게임 중 최장 거점 방어 구간
- Beacon (0, 18) — 맵 정북 최외곽 (radius=13 원보다 5m 추가). B3(0,13)에서 직선 5m
- 최적 루트 (HoldOut 우선): B2 → B3 → Beacon 진입 → HoldOut 중 S1/S2/SP1 부분 수집
- 전체 수집 루트 (3성 도전): B1 → B2 → B3 → S1 → S2 → S3 → SP1 → SP2 → Beacon
  - 총 이동 거리: ~120m → 150초 기준 매우 빡빡 (3성 조건: 150×0.45 = 67.5초)
- 다중 목표 압박: 긴 원정 + HoldOut 90초 + 복합 지형
- 환경 위험: 해안 절벽, 야간 파도, 다중 목표 압박, 복합 지형

**난이도별 변화**

| 항목 | 값 |
|------|----|
| 2성 조건 | targetPickupCount×0.65 = 3.9 → 4개 이상 수집 |
| 3성 조건 | 150초 × 0.45 = 67.5초 이내 완료 (HoldOut 90초 포함 — 사실상 HoldOut만으로 시간 초과 → 3성은 빠른 수집 필수) |
| HoldOut | 90초 — 최장 거점 방어 |

> **설계 노트**: 3성 조건 (67.5초)이 HoldOut (90초)보다 짧으므로, 3성을 달성하려면 수집을 전혀 하지 않고 즉시 Beacon으로 달려 HoldOut을 시작해야 함. 수집 vs 속도 간 전략적 선택이 핵심.

## 4. 픽업 배치 공식

```
position(i) = (cos(2π × i / total) × radius, 0.75, sin(2π × i / total) × radius)
```

파라미터:

| 변수 | 정의 |
|------|------|
| `i` | 픽업 인덱스 (0-based) |
| `total` | bloomPickupCount + scrapPickupCount + seedPodPickupCount |
| `radius` | district.pickupSpawnRadius |

인덱스 순서: BloomDust → Scrap → SeedPod (코드 구현 순서 따름)

**배치 요약표**

| 지구 | total | radius | 각도 간격 |
|------|-------|--------|-----------|
| Dock | 4 | 7m | 90.0° |
| Reed Fields | 6 | 9m | 60.0° |
